1. Introduction

Welcome to this comprehensive tutorial on Selenium-based automation framework! This guide will teach you everything you need to know about test automation using:

  • Selenium WebDriver - For browser automation
  • Cucumber - For writing tests in plain English (BDD)
  • JUnit - For running tests
  • Page Object Model (POM) - For organizing code
  • Maven - For managing dependencies

2. Framework Architecture

Our framework uses a layered architecture where each layer has a specific responsibility:

automation-project/ ├── src/test/ │ ├── java/com/company/ │ │ ├── context/ # Sharing data between tests │ │ ├── hooks/ # Setup and cleanup actions │ │ ├── pages/ # Page Object classes (one per page) │ │ ├── runners/ # Test execution configuration │ │ ├── stepDefinitions/ # Test step implementations │ │ └── utils/ # Helper classes (waits, config, etc.) │ └── resources/ │ ├── config/ # Configuration files │ └── features/ # Test scenarios in plain English ├── pom.xml # Dependencies and build settings └── logs/ # Test execution logs

Design Patterns Used

Page Object Model (POM)
Each web page = one Java class. It separates the location of the element (locators) from the test logic.
Dependency Injection
Allows sharing state (data) seamlessly between different test steps without static variables.
Singleton Pattern
Ensures only one instance of configuration or driver exists per thread context.
Factory Pattern
Used by PageFactory to initialize web elements efficiently when pages are loaded.

5. Framework Components Explained

1. TestContext - Sharing Data Between Steps

Location: src/test/java/com/company/context/TestContext.java
Allows different test steps to share information (e.g., created user ID).

// STORING data (in any step)
testContext.setContext("USER_EMAIL", "[email protected]");
testContext.setContext("USER_ID", 12345);
testContext.setContext("IS_VERIFIED", true);

// RETRIEVING data (in another step)
String email = testContext.getContextAsString("USER_EMAIL");
Integer userId = testContext.getContextAsInteger("USER_ID");
Boolean verified = testContext.getContextAsBoolean("IS_VERIFIED");

// CHECKING if data exists
if (testContext.containsContext("USER_EMAIL")) {
    // Do something
}

// CLEARING all data (done automatically after each test)
testContext.clearContext();

@Before
public void beforeScenario(Scenario scenario) {
    logger.info("Starting test: " + scenario.getName());
    
    // Open browser
    DriverManager.initializeDriver();
    WebDriver driver = DriverManager.getDriver();
    context.setDriver(driver);
    
    // Go to application

}

@After
public void afterScenario(Scenario scenario) {
    logger.info("Test finished: " + scenario.getName());
    
    // Take screenshot if failed
    if (scenario.isFailed()) {
        takeScreenshot(scenario);
    }
    
    // Close browser
    DriverManager.quitDriver();
}

5. WaitHelper - Waiting for Elements

Location: src/test/java/com/company/utils/WaitHelper.java
Waits for elements to be ready. Critical for dynamic web pages.

// Wait until element is visible on screen
waitHelper.waitForElementVisible(element);

// Wait until element is clickable
waitHelper.waitForElementClickable(element);

// Wait until element exists in HTML
waitHelper.waitForElementPresent(element);

// Check if element is displayed (returns true/false)
boolean isVisible = waitHelper.isElementDisplayed(element);

// Wait with custom timeout (in seconds)
waitHelper.waitForElement(element, 30);

// Hard wait - pause for specific time (USE SPARINGLY!)
waitHelper.hardWait(2000); // 2 seconds

Step 2: Create the Page Object

Location: src/test/java/com/company/pages/LoginPage.java

package com.company.pages;

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.support.FindBy;
import org.openqa.selenium.support.PageFactory;

public class LoginPage extends BasePage {

    // ========== Define Web Elements ==========
    @FindBy(id = "email")
    private WebElement emailInput;
    
    @FindBy(id = "password")
    private WebElement passwordInput;
    
    @FindBy(xpath = "//button[@type='submit']")
    private WebElement loginButton;
    
    @FindBy(css = ".error-message")
    private WebElement errorMessage;
    
    @FindBy(xpath = "//h1[contains(text(),'Welcome')]")
    private WebElement welcomeMessage;

    // ========== Constructor ==========
    public LoginPage(WebDriver driver) {
        super(driver);  // Call parent class constructor
        PageFactory.initElements(driver, this);  // Initialize all @FindBy elements
    }

    // ========== Page Actions (Methods) ==========
    public boolean isLoginPageDisplayed() {
        try {
            return waitHelper.isElementDisplayed(loginButton);
        } catch (Exception e) {
            return false;
        }
    }
    
    public void enterEmail(String email) {
        waitHelper.waitForElementVisible(emailInput);
        enterText(emailInput, email);
    }
    
    public void enterPassword(String password) {
        waitHelper.waitForElementVisible(passwordInput);
        enterText(passwordInput, password);
    }
    
    public void clickLoginButton() {
        waitHelper.waitForElementClickable(loginButton);
        click(loginButton);
    }
    
    public String getErrorMessage() {
        waitHelper.waitForElementVisible(errorMessage);
        return getText(errorMessage);
    }
    
    public boolean isWelcomeMessageDisplayed() {
        try {
            return waitHelper.isElementDisplayed(welcomeMessage);
        } catch (Exception e) {
            return false;
        }
    }
    
    public void login(String email, String password) {
        enterEmail(email);
        enterPassword(password);
        clickLoginButton();
    }
}

Step 3: Create Step Definitions

Location: src/test/java/com/company/stepDefinitions/LoginSteps.java

package com.company.stepDefinitions;

import com.company.context.TestContext;
import com.company.pages.LoginPage;
import com.company.utils.ConfigReader;
import io.cucumber.java.en.Given;
import io.cucumber.java.en.Then;
import io.cucumber.java.en.When;
import org.junit.Assert;
import org.openqa.selenium.WebDriver;

public class LoginSteps {

    private WebDriver driver;
    private TestContext testContext;
    private LoginPage loginPage;
    private ConfigReader configReader;

    // Constructor - TestContext is automatically injected by Cucumber
    public LoginSteps(TestContext testContext) {
        this.testContext = testContext;
        this.driver = testContext.getDriver();
        this.loginPage = new LoginPage(driver);
        this.configReader = new ConfigReader();
    }

    @Given("the login page is open")
    public void the_login_page_is_open() {
        String baseUrl = configReader.getBaseUrl();
        driver.get(baseUrl + "/login");
        Assert.assertTrue("Login page should be displayed", loginPage.isLoginPageDisplayed());
    }

    @When("I enter email {string}")
    public void i_enter_email(String email) {
        loginPage.enterEmail(email);
        testContext.setContext("USER_EMAIL", email);
    }

    @When("I enter password {string}")
    public void i_enter_password(String password) {
        loginPage.enterPassword(password);
    }

    @When("I click the login button")
    public void i_click_the_login_button() {
        loginPage.clickLoginButton();
    }

    @Then("I should be redirected to the dashboard")
    public void i_should_be_redirected_to_the_dashboard() {
        loginPage.waitHelper.hardWait(2000);
        String currentUrl = driver.getCurrentUrl();
        Assert.assertTrue("Should redirect to dashboard", currentUrl.contains("/dashboard"));
    }

    @Then("I should see welcome message")
    public void i_should_see_welcome_message() {
        Assert.assertTrue("Welcome message should be displayed", loginPage.isWelcomeMessageDisplayed());
    }

    @Then("I should see error message {string}")
    public void i_should_see_error_message(String expectedMessage) {
        loginPage.waitHelper.hardWait(1000);
        String actualMessage = loginPage.getErrorMessage();
        Assert.assertTrue("Error message should contain: " + expectedMessage, 
            actualMessage.contains(expectedMessage));
    }

    @Then("I should remain on the login page")
    public void i_should_remain_on_the_login_page() {
        String currentUrl = driver.getCurrentUrl();
        Assert.assertTrue("Should remain on login page", currentUrl.contains("/login"));
    }
}

7. Best Practices

DO THIS

// Good: Descriptive method names
public void clickLoginButton() { }

// Good: Combine related actions
public void login(String email, String password) {
    enterEmail(email);
    enterPassword(password);
    clickLoginButton();
}

// Good: Explicit waits
waitHelper.waitForElementVisible(element);

DON'T DO THIS

// Bad: Exposing WebElements
public WebElement getLoginButton() { }

// Bad: Assertions in Page Objects
public void verifyLogin() {
    Assert.assertTrue(isDashboardDisplayed());
}

// Bad: Hard waits
Thread.sleep(5000);

Locator Strategy (Priority Order)

  1. ID (Best): @FindBy(id = "email")
  2. Name (Good): @FindBy(name = "username")
  3. CSS Selector (Fast): @FindBy(css = ".error-message")
  4. XPath (Last Resort): @FindBy(xpath = "//button[text()='Login']")

9. Common Errors & Solutions

❌ Error 1: NoSuchElementException

Cause: Selenium cannot find the element on the page.

  • Not loaded yet: Use waitHelper.waitForElementVisible(element).
  • Wrong locator: Check ID/XPath in DevTools.
  • Inside Iframe: Switch to iframe: driver.switchTo().frame("name").
  • Not visible: Scroll to element first.
❌ Error 2: ElementNotInteractableException

Cause: Element exists but cannot be clicked.

  • Not clickable yet: Use waitHelper.waitForElementClickable(element).
  • Covered by overlay: Wait for overlay to disappear.
  • Disabled: Check element.isEnabled().
❌ Error 3: StaleElementReferenceException

Cause: Page refreshed/changed, making reference invalid.

  • Page refreshed: Re-find element after refresh.
  • Dynamic DOM: Use @FindBy (re-finds automatically).
  • Removed/Added: Wait and re-find.
❌ Error 4: TimeoutException

Cause: Element didn't appear within wait time.

  • Too slow: Increase timeout waitHelper.waitForElement(el, 60).
  • Wrong locator: Verify element manually.
  • App issue: Element never appears.
  • Network: Increase global timeouts in config.
❌ Error 5: WebDriverException

Cause: Browser/Driver issues.

  • Browser not installed.
  • Driver not downloaded (use WebDriverManager).
  • Version mismatch (update browser or driver).
  • Permission issues (Linux/Mac chmod +x).
❌ Error 6: NullPointerException

Cause: Using null object.

  • Element not init: Missing PageFactory.initElements(driver, this).
  • Driver null: Initialize driver before use.
  • Context null: Check if data exists before retrieval.
❌ Error 7: InvalidSelectorException

Cause: Bad XPath/CSS syntax.

  • XPath: //button[text()='Login'] (Need quotes).
  • CSS: button[type='submit'] (Need quotes).
❌ Error 11: BUILD FAILURE

Cause: Maven build failed.

  • Java Version: Must be 11+.
  • Dependency fail: Delete .m2 folder and run mvn clean install -U.
  • Compilation error: Fix syntax errors in code.
  • Port in use: Kill java processes.

10. Troubleshooting Guide

General Debugging Steps

  1. Check Logs: Look in logs/application.log for ERRORs.
  2. Take Screenshots: Check target/screenshots/.
  3. Add Logging: Print variable states to console/log.
  4. Pause Execution: Add temporary waitHelper.hardWait(5000).
  5. Debug Mode: Use IDE breakpoints to step through code.

Quick Fixes Checklist

  • Add explicit wait before action
  • Verify element locator in DevTools
  • Check if element is in iframe
  • Scroll to element before clicking
  • Use JavaScript click if normal click fails
  • Increase timeout in config.properties
  • Run in non-headless mode to see execution

11. FAQ

Q1: How do I find element locators?
Right-click element -> Inspect -> Right-click HTML -> Copy -> Copy Selector/XPath. Verify in console with $x("//xpath") or $$("css").
Q2: How do I handle dropdowns? 
DropdownHelper dd = new DropdownHelper(driver);
dd.selectByVisibleText(element, "Option 1");
dd.selectByValue(element, "value1");
Q3: How do I handle alerts/popups? 
AlertHelper alert = new AlertHelper(driver);
alert.acceptAlert(); // Click OK
alert.dismissAlert(); // Click Cancel
Q4: How do I upload files? 
@FindBy(id = "fileInput") WebElement fileInput;
fileInput.sendKeys("C:\\path\\to\\file.pdf");
Q5: How do I handle multiple windows/tabs? 
String main = driver.getWindowHandle();
// ... open new window ...
for(String win : driver.getWindowHandles()) {
    if(!win.equals(main)) driver.switchTo().window(win);
}
Q6: How do I generate random data? 
String uniqueEmail = "test_" + System.currentTimeMillis() + "@example.com";
Q7: How do I run tests in parallel?
Configuring `maven-surefire-plugin` in `pom.xml` with `methods` and `3`.
Q8: How do I wait for page load?
Wait for a specific element (best) or use JS: `return document.readyState == 'complete'`.
Q9: How do I debug failing tests?
Run in non-headless mode, add breakpoints, inspect logs/screenshots.
Q10: How do I handle dynamic elements?
Use XPaths with `contains()` or format strings: `String.format("//button[text()='%s']", btnName)`.
